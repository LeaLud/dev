Git

Definición

Version control system. Sistema de control de versiones. Documentación oficial

Comandos básicos de consola de Git

Configuración inicial

  1. git init dentro del directorio de trabajo para ejecutar git
  2. git config --global user.name "USUARIO" configura el usuario de git en toda la computadora
  3. git config --global user.email "EMAIL" configura el email de usuario en toda la computadora

Gestión de ramas

  1. git branch lista las ramas existentes e indica la actual
  2. git branch NOMBRE crea una nueva la rama
  3. git branch -m NOMBRE cambia el nombre por defecto de la rama
  4. git switch NOMBRERAMA cambia a la rama señalada
  5. git checkout NOMBRERAMA cambia a la rama señalada solo si se aplicó un commit
  6. git merge NOMBRERAMA fusiona la rama actual con la rama señalada. Ubicarse en la que absorberá los cambios de la otra
  7. git branch -d NOMBRERAMA elimina la rama seleccionada mientras no se esté en ella

Gestión de commits

  1. git add ARCHIVO.EXTENSION | CARPETA prepara para el commit los archivos y carpetas indicados. Se pueden señalar varios elementos separando los nombres con espacios
  2. git add . prepara para el commit todos los archivos y carpetas que se han modificado
  3. git commit -m "NOMBRE" aplica un commit sobre el archivo. Este nombre genera un ID único
  4. git restore --staged ARCHIVO.EXTENSION quita el archivo indicado de la zona de commit
  5. git restore --staged . quita todos los elementos de la zona de commit
  6. git checkout ARCHIVO.EXTENSION | CARPETA restaura el último commit de un archivo o contenido de una carpeta
  7. git restore ARCHIVO.EXTENSION | CARPETA restaura el último commit de un archivo o contenido de una carpeta
  8. git restore --staged ARCHIVO.EXTENSION recupera el último commit eliminado con rm --cached
  9. git reset muestra los archivos que pueden recuperar una versión anterior
  10. git reset --hard IDCOMMIT elimina/devuelve todos los commits posteriores/anteriores desde el commit indicado

Gestión de tags

  1. git tag NOMBRE crea una etiqueta que identifica un commit. Esta etiqueta puede ser usada en lugar del ID del commit
  2. git tag lista el registro de tags
  3. git checkout NOMBRETAG cambia a la etiqueta señalada

Gestión de stash

  1. git stash almacena temporalmente los cambios no guardados con commit
  2. git stash pop restaura los cambios almacenados temporalmente y que no han sido guardados con commit
  3. git stash drop elimina el último stash junto con los cambios almacenados temporalmente en él
  4. git stash list muestra una lista de los stash creados. Una vez que un stash es restaurado o eliminado, no se muestra en esta lista

Gestión de archivos y carpetas

  1. git mv ARCHIVO.EXTENSION | CARPETA mueve o renombra un archivo o carpeta
  2. git rm ARCHIVO.EXTENSION elimina físicamente un archivo y lo agrega al área de espera
  3. git rm --cached ARCHIVO.EXTENSION elimina el archivo del commit pero no físicamente
  4. git rmdir CARPETA elimina una carpeta

Ignorando archivos

  1. .gitignore es el archivo donde se listan archivos y carpetas que serán ignorados al ser modificados y no se mostrarán en el git status. Se puede crear con el comando touch
  2. ARCHIVO.EXTENSION dentro de .gitignore para listar archivos a ignorar
  3. CARPETA dentro de .gitignore para listar carpetas a ignorar
  4. CARPETA/SUBCARPETA dentro de .gitignore para listar subcarpetas a ignorar

Revisión de logs

  1. git status muestra el estado de los archivos
  2. git checkout muestra el estado de los archivos
  3. git log muestra el registro de commits creados hasta el momento. Para salir de esa pantalla presionar la Q
  4. git reflog muestra el registro de commits creados y eliminados
  5. git diff muestra los cambios realizados a partir del último commmit
  6. git diff NOMBRERAMA muestra los cambios realizados a partir del último commmit en la rama indicada
  7. git checkout muestra archivos y carpetas que han sido modificados y no fueron guardados con commit

Integración remota

  1. Generación de una nueva clave SSH y adición al agente SSH (Windows). Ruta: C:\Program Files\Git\etc\ssh
  2. git fetch comprueba si existen cambios en el repositorio remoto
  3. git pull comprueba si existen cambios en el repositorio remoto y los copia en el repositorio local
  4. git push -u origin NOMBRERAMA sube los cambios desde el repositorio local al remoto
  5. git clone URLREPOSITORIO clona un repositorio remoto a la ubicación local desde donde se ejecutó el comando

Subir un repositorio local con URL del repositorio remoto

  1. git remote add origin https://github.com/USUARIO/REPOSITORIO.git
  2. git branch -M main
  3. git push -u origin main

Comandos básicos de la consola GNU Bash (Bourne-again shell)

  1. cd muestra la ruta del directorio actual
  2. cd RUTA | CARPETA cambia de directorio
  3. cd .. sube un nivel de directorio
  4. ls lista la estructura de directorios actual
  5. ls CARPETA/ lista la estructura de directorios de la carpeta
  6. dir lista la estructura de directorios
  7. touch ARCHIVO.EXTENSION crea un archivo o carpeta
  8. mkdir CARPETA crea una carpeta
  9. rm ARCHIVO.EXTENSION borra un archivo o carpeta
  10. rmdir CARPETA borra una carpeta
  11. rm -r CARPETA borra una carpeta
  12. mv ARCHIVO.EXTENSION NUEVOARCHIVO.EXTENSION | CARPETA NUEVACARPETA | ARCHIVO.EXTENSION RUTA | CARPETA RUTA mueve o cambia el nombre al archivo o carpeta
  13. copy ARCHIVO.EXTENSION | CARPETA copia el archivo a la carpeta o ruta indicada. Usar ../ para copiar en un nivel superior
  14. move ARCHIVO.EXTENSION | CARPETA mueve el archivo a la carpeta o ruta indicada. Usar ../ para mover a un nivel superior
  15. history muestra los últimos comandos ejecutados junto con un número que permite volver a repetir el comando
  16. reset resetea el HEAD actual al estado indicado
  17. clear limpia la consola
  18. exit cierra la consola
